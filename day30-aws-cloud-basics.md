# Day 30 - AWS Cloud Basics

## What is AWS?

AWS (Amazon Web Services) is a cloud computing platform provided by Amazon.

It offers on-demand computing services such as:

- Storage
- Databases
- Virtual Servers
- Networking
- Machine Learning
- Analytics

AWS follows a pay-as-you-go pricing model.

---

## Why AWS?

Instead of buying expensive physical servers, companies rent cloud resources only when needed.

Benefits:

- High Availability
- Scalability
- Security
- Global Infrastructure
- Cost Effective

---

## What is Cloud Computing?

Cloud computing is the delivery of computing services such as servers, storage, databases, networking, and software over the internet.

---

## AWS Global Infrastructure

AWS consists of:

- Regions
- Availability Zones (AZs)
- Edge Locations

### Region

A Region is a physical geographic location where AWS has data centers.

Examples:

- US East (N. Virginia)
- US West (Oregon)
- Asia Pacific (Mumbai)

---

### Availability Zone

An Availability Zone (AZ) is one or more isolated data centers within a Region.

Benefits:

- High Availability
- Fault Tolerance

Example:

US East (N. Virginia)

├── AZ A

├── AZ B

└── AZ C

---

## AWS Shared Responsibility Model

AWS is responsible for:

- Physical security
- Hardware
- Networking
- Data center operations

Customer is responsible for:

- Data
- User access
- Passwords
- Applications
- Operating systems (depending on the service)

---

## Benefits of AWS

- Elasticity
- Scalability
- Reliability
- Global Availability
- Pay-as-you-go pricing

## Interview Questions

### Q1. What is AWS?

Answer:

AWS (Amazon Web Services) is a cloud computing platform that provides on-demand IT resources over the internet.

---

### Q2. What is a Region?

Answer:

A Region is a physical geographic area where AWS has multiple data centers.

---

### Q3. What is an Availability Zone?

Answer:

An Availability Zone is one or more isolated data centers within an AWS Region.

---

### Q4. What is the Shared Responsibility Model?

Answer:

AWS secures the cloud infrastructure, while customers are responsible for securing their data, applications, and user access.
